# ADR-0001: ADR の置き場、書式、番号ルール

- 状態: 採択
- 日付: 2026-07-08

## 背景

開発運用の判断が会話、PR 本文、Issue コメントに分散すると、次のサブエージェントやメインスレッドが同じ判断を再現できない。
特に GitHub 運用、Dependabot、PR レビュー、Issue close の方針は、実装とは別に継続して参照できる形で残す必要がある。

## 決定

- ADR は `docs/adr/` に保存する。
- ADR の索引は `docs/adr/README.md` とする。
- ファイル名は `NNNN-kebab-case-title.md` とし、番号は 4 桁の連番にする。
- 番号は採番後に再利用しない。削除された方針や置換された方針があっても番号を詰めない。
- ADR 本文は次の見出しを基本形にする。
  - `背景`
  - `決定`
  - `影響`
  - `関連`
- 状態は `提案中`、`採択`、`置換済み`、`却下` のいずれかを使う。
- 採択済み ADR の意味を後から変える場合は、原則として新しい ADR を追加し、旧 ADR から新 ADR へリンクする。
- 誤字、リンク切れ、表現の明確化など、方針の意味を変えない修正は既存 ADR へ直接反映してよい。

## 影響

- 新しい運用判断は docs の通常文書だけでなく、必要に応じて ADR として残す。
- `README.md` や既存 docs は、詳細方針を重複して持たず、ADR へのリンクで正を示す。
- PR レビューでは、運用判断が ADR に残っているかを確認できる。
