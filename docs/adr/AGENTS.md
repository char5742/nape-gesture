# docs/adr/AGENTS.md

この階層は ADR の正本です。既存の採択済み ADR の意味を静かに変えず、方針変更は追跡できる形で残してください。

## ADR ルール

- 新しい ADR は `NNNN-kebab-case-title.md` とし、番号は 4 桁の連番にする。
- 番号は採番後に再利用しない。
- 本文は原則として `背景`、`決定`、`影響`、`関連` を持つ。
- 状態は `提案中`、`採択`、`置換済み`、`却下` のいずれかにする。
- 採択済み ADR の意味を変える場合は、新しい ADR で置き換え理由を明記し、旧 ADR からリンクする。
- ADR を追加したら `README.md` の一覧更新が必要か確認する。ただし、既に別作業が `README.md` を編集中なら競合しない形で扱う。
