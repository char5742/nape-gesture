# ゴール要件

## 最終ゴール

Nape Proなどの通常マウス入力を、macOS上でトラックパッド級のジェスチャー操作に変換する常駐ツールを作る。event contractとパラメータはApple公式資料、Apple OSS、このリポジトリで取得した純正trackpad / Nape Proログを正本とし、採用したfield、状態遷移、係数、調整値を計測証跡まで追跡可能にする。

## 必須機能

- 対象デバイスを識別し、通常のマウス操作とジェスチャー操作を安全に切り分ける
- 対象デバイス識別は通常のマウス用途だけに限定せず、複合 HID や特殊 usage として露出する場合も候補に含める
- ボタン押下、移動量、ホイール、時間、速度、方向を低遅延で取得する
- 特定ボタン押下中のドラッグやホイールをジェスチャーとして認識する
- ジェスチャーボタン未押下時は、通常マウス入力を改変しない
- ジェスチャー成立後は、元のボタン押下や移動イベントがアプリへ漏れて誤動作しない
- ボタンを離した瞬間に確実に通常マウス状態へ戻る
- ピクセル単位のなめらかな縦横スクロールを生成する
- scroll lifecycleの`began` / `changed` / `ended` / `cancelled`と、別系列のmomentum `began` / `continued` / `ended`を扱う
- 慣性、加速度、しきい値、方向ロック、キャンセル条件を独自に調整できる
- Mission Control、Spaces 移動、ページ戻る/進む、ズーム、横スクロールを割り当て可能にする
- 誤爆時に即座に停止できる `Control + Option + Command + G` の一方向キルスイッチを持つ
- アクセシビリティ権限など必要権限の状態を検出し、復旧導線を出す
- `.app` は Dock に表示される通常 GUI アプリとして起動し、起動時に設定ウィンドウを開く

## 不要な機能

- アプリごとの有効・無効
- アプリごとの感度設定
- アプリごとのジェスチャー割り当て

特定ボタンを押していない間は通常マウスとして振る舞うため、アプリ別制御ではなく、デバイス全体の操作体系として完成させる。

## 操作要件

- ジェスチャーボタン未押下時は、クリック、ドラッグ、ホイール、サイドボタンなどを通常通り通す
- ジェスチャーボタン押下中のみ、移動量・ホイール・時間・速度・方向を解釈する
- 微小な揺れではジェスチャーを開始しない
- 一定しきい値を超えた時点でジェスチャーとして確定する
- 確定後は方向ロックやキャンセル条件に従い、意図しない入力を抑制する
- 加速度は速度しきい値、指数、最大倍率として調整できる
- キャンセル条件は最大継続時間、無入力時間、方向ロック後の軸ずれ比として調整できる
- ボタンを離したら `ended` または `cancelled` として処理し、必要なら慣性に移行する
- 慣性終了後は必ず通常状態へ戻る

## Spaces / Mission Control 要件

完成形では、`Ctrl + ←/→`などのkeyboard shortcutをgesture代替にしない。純正trackpad driverが上位へ出すscroll / gesture、DockSwipe、NavigationSwipe、magnification event contractを再導出し、macOSの標準gesture処理へsystem-wideに渡す。

完了条件:

- 純正トラックパッド操作時のイベントログを取得できる
- Nape Pro 操作時の入力ログを取得できる
- 生成イベントのログを同じ形式で比較できる
- Spaces 移動や Mission Control で、画面挙動・イベント列・体感差分を検証できる
- event type、subtype、phase、momentum、field、timestamp、順序を純正trackpad logと比較できる
- 通常SDK非公開のcontractはcompatibility adapterへ隔離し、macOS versionごとの差分と破損時のfail-closed条件を説明できる
- supported contractは登録済みfixture ID、SHA-256、schema、contract ID、OS version / build、fixture実体の完全一致を機械判定できる
- AX、対象PID配送、application別分岐、keyboard shortcut fallbackを製品runtimeに持たない

## 検証ツール要件

- `Event Logger`: 実デバイス、純正トラックパッド、生成イベントを同じ形式で記録。時間制限、出力先指定、生成イベントの除外/抽出を指定できる
- `Log Comparator`: 純正入力ログと生成イベントログを比較し、フェーズ分布、precise 率、スクロール総量の差を説明
- `HID Log Analyzer`: Nape Pro などの生 HID 入力を device / usage ごとに集計し、対象 usage と値域を特定
- `Event Generator`: 任意のスクロール・フェーズ・慣性イベントを発火
- `Reference Target App`: `scrollWheel`、`swipe`、`magnify` などを受け取り差分表示し、AppKit 受信イベントを JSON Lines として保存・集計
- `System Behavior Test`: Finder、Safari、Mission Control、Spaces で実挙動を確認し、dry-run 時に生成予定イベントを JSON Lines として保存できる
- `Benchmark`: 認識器とスクロール計画の純粋ロジック処理時間を測定
- `Doctor`: 権限、対象デバイス、HID 入力監視、ベンチマーク、復旧手順を一括確認
- `Verification Guide`: 完成判定に必要な証跡、実機検証手順、既知の失敗条件と回復手順を `docs/verification.md` に集約

## 品質要件

- 入力遅延が体感できない水準であること
- CPU 使用率が常駐アプリとして許容範囲であること
- 純粋ロジックのベンチマーク値を記録し、実機イベントタップ計測と分けて判断できること
- 通常クリック、通常ドラッグ、通常ホイール操作を壊さないこと
- ジェスチャー中に入力ループや暴走が起きないこと
- スリープ復帰、デバイス抜き差し、権限変更後も復旧すること
- macOS 更新で壊れやすい箇所を分離すること
- JSON を直接編集した場合も、不正な感度、慣性、キャンセル、対象デバイス条件を起動前に検出すること
- 主要ロジックに自動テストを用意すること

## ライセンス要件

- 実装contractとパラメータはApple公式資料、Apple OSS、自前ログから再導出する
- field番号、関数名、定数、状態遷移、係数、調整値を、採用根拠となる資料または自前ログまで追跡できるようにする
- 製品surfaceの固有名は、Nape Gesture自身の仕様と、実際の依存関係・法定通知に必要なものへ限定する
- 実際に第三者依存または派生物を追加する場合は、そのライセンスが要求する固有名、著作権表示、通知を省略しない
- 配布する場合は独自ライセンスと依存ライセンスを明記する

## 完成判定

- 常駐アプリとして日常利用できる
- 設定 UI から主要ジェスチャーを調整できる
- `.app` 起動時に Dock 表示、設定ウィンドウ、メニューバー常駐 UI が確認できる
- イベントログで純正入力との差分を説明できる
- Spaces / Mission Control まわりの限界が実測ベースで明確
- 既知の失敗条件と回避策が文書化されている
- ビルド、テスト、権限導線、配布物まで整っている
